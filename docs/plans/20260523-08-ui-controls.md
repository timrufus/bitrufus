# Plan: Pause, Resume, and Remove Controls

## Overview

Give the user direct control over each torrent. A context menu on each row toggles pause/resume, and a remove action prompts for whether to delete the on-disk files before doing so.

## Validation Commands

- `xcodebuild -project apps/TorrentApp.xcodeproj -scheme TorrentApp -configuration Debug build`
- Manual: pause an active row — speed falls to zero, state badge switches to Paused. Resume — speed climbs again. Remove with "Also delete files" off — row disappears, files remain. Remove with the toggle on — row and files both gone.

### Task 1: Context menu actions

- [ ] Attach `.contextMenu` to `TorrentRow` showing either `Pause` or `Resume` based on `stats.state`, plus `Remove…`.
- [ ] Wire pause/resume to `engine.pause(id:)` and `engine.resume(id:)`; never block the main thread on these calls.
- [ ] Let the next poll tick reflect the new state via the existing 500 ms refresh — no manual UI nudge.

### Task 2: Remove confirmation dialog

- [ ] On `Remove…`, present a `.confirmationDialog` with a "Also delete files from disk" toggle defaulting to off.
- [ ] On confirm, call `engine.remove(id:, deleteFiles:)` and remove the VM from `store.torrents`.
- [ ] On cancel, do nothing.

### Task 3: Error handling

- [ ] Surface engine errors from pause/resume/remove via a non-blocking alert (e.g. an `.alert` bound to a `lastError` property on the store).
- [ ] Confirm operations on an already-paused or already-removed torrent fail gracefully without crashing.
- [ ] Add a debug log line for any engine error so manual testing can spot regressions.
